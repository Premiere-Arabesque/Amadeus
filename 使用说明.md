# Amadeus 使用说明

## 1. 项目简介

Amadeus 是一个以人格为核心、事件驱动的持续运行型 Agent Runtime。

当前这版已经可以跑通下面这条主链路：

- 初始化人格
- 接收用户消息
- 进行计划、执行、重规划
- 写入记忆
- 通过 API 查看运行状态与记忆状态

如果没有配置外部大模型，系统也可以用启发式 fallback 跑最小闭环；如果配置了模型 API，则会启用真实模型能力。

## 2. 当前已接好的能力

- 多模型路由：`dialogue / decision / memory`
- `PydanticAI` 兼容的大模型调用
- 语义检索 + BM25 + LLM reranker
- 阿里云 DashScope 兼容的 semantic embedding
- Core Memory / Active Memory / Raw Log / Snapshot
- 内置 `read_url` 和 `search_web` 两个 Real Zone 能力
- FastAPI 调试与运行接口

当前还没有真正接入的是：

- `emotional_embedding`
- 完整版 `Archive Memory`

## 3. 环境要求

- Python `3.12`
- 推荐使用项目里的虚拟环境，或使用 `uv`

如果你已经在当前项目目录里看到 `.venv`，可以直接复用它。

## 5. 配置 `.env`

先把 [.env.example](C:/Users/lenovo/Desktop/agent/Amadeus/.env.example) 复制成 `.env`。

最小可运行配置有两种：

- 不填模型配置：系统走启发式 fallback
- 填大模型配置：系统启用真实模型调用

### 5.1 阿里云 Qwen + DashScope Embedding 示例

如果你使用阿里云 Qwen 作为主模型，`text-embedding-v4` 作为语义向量模型，可以这样填：

```dotenv
AMADEUS_DIALOGUE_PROVIDER=alibaba
AMADEUS_DIALOGUE_MODEL=qwen-plus
AMADEUS_DIALOGUE_API_KEY=你的DashScopeKey

AMADEUS_DECISION_PROVIDER=alibaba
AMADEUS_DECISION_MODEL=qwen-plus
AMADEUS_DECISION_API_KEY=你的DashScopeKey

AMADEUS_MEMORY_PROVIDER=alibaba
AMADEUS_MEMORY_MODEL=qwen-turbo
AMADEUS_MEMORY_API_KEY=你的DashScopeKey

AMADEUS_MEMORY_SEMANTIC_EMBEDDING_PROVIDER=alibaba
AMADEUS_MEMORY_SEMANTIC_EMBEDDING_MODEL=text-embedding-v4
AMADEUS_MEMORY_SEMANTIC_EMBEDDING_API_KEY=你的DashScopeKey

AMADEUS_MEMORY_RETRIEVAL_SEMANTIC_ENABLED=true
AMADEUS_MEMORY_RETRIEVAL_BM25_ENABLED=true
AMADEUS_MEMORY_RETRIEVAL_EMOTIONAL_ENABLED=false
AMADEUS_MEMORY_RETRIEVAL_RERANKER_ENABLED=true

AMADEUS_MEMORY_ACTIVE_RETENTION_DAYS=999
AMADEUS_EXECUTION_MAX_INNER_LOOP_TURNS=7
AMADEUS_EXECUTION_LOOP_PRE_REPLAN_BUFFER_SECONDS=30
```

补充说明：

- `AMADEUS_MEMORY_SEMANTIC_EMBEDDING_PROVIDER` 不填时，默认继承 `AMADEUS_MEMORY_PROVIDER`
- `AMADEUS_MEMORY_SEMANTIC_EMBEDDING_API_KEY` 不填时，默认继承 `AMADEUS_MEMORY_API_KEY`
- `alibaba` 的 semantic embedding 路由在不填 `BASE_URL` 时，会自动使用 DashScope 兼容地址

更完整的配置说明见 [docs/configuration.md](C:/Users/lenovo/Desktop/agent/Amadeus/docs/configuration.md)。

## 7. 推荐使用顺序

建议按这个顺序使用：

1. 先检查健康状态
2. 初始化人格
3. 发送消息驱动系统进入运行
4. 查看 runtime state 和 memory state
5. 按需使用 `run-once`、`pause/resume`、`clock` 相关接口调试

## 8. 主要接口

这些接口都定义在 [app/main.py](C:/Users/lenovo/Desktop/agent/Amadeus/app/main.py)。

### 8.1 健康检查

```http
GET /health
```

用途：

- 看服务是否活着
- 看 scheduler 是否在运行
- 看 runtime 是否处于暂停状态

### 8.2 初始化人格

```http
POST /api/persona/bootstrap
```

示例请求：

```json
{
  "name": "Kurisu",
  "seed_text": "A sharp, curious researcher who likes quiet routines and careful planning."
}
```

### 8.3 获取当前人格

```http
GET /api/persona
```

### 8.4 发送用户消息

```http
POST /api/messages
```

示例请求：

```json
{
  "user_id": "user-1",
  "channel": "api",
  "text": "今晚想做什么？"
}
```

这是最常用的入口。它会：

- 写入事件
- 推进一次运行循环
- 返回本轮 outcome、外发消息和最新 runtime state

### 8.5 手动推进一次运行

```http
POST /api/runtime/run-once
```

适合没有新消息时手动触发一轮 runtime。

### 8.6 暂停和恢复后台调度

```http
POST /api/runtime/lifecycle/pause
POST /api/runtime/lifecycle/resume
```

### 8.7 查看 runtime 状态

```http
GET /api/runtime/state
```

这里能看到：

- 当前计划状态
- 下一次唤醒时间
- 最新快照
- 当前 scheduler 状态

### 8.8 查看记忆状态

```http
GET /api/memory?limit=10
```

这里能看到：

- Core Memory
- 最近的 Active Memory
- 最近的 Archive Memory
- 最近的 Raw Log

### 8.9 搜索记忆

```http
GET /api/memory/search?query=xxx&top_k=5
```

### 8.10 可控时钟与场景回放

```http
POST /api/runtime/clock/set
POST /api/runtime/clock/advance
POST /api/runtime/scenario/run
```

这些接口更适合测试和调试，不是日常主入口。

```powershell
  -H "Content-Type: application/json" `
  -d "{\"user_id\":\"user-1\",\"channel\":\"api\",\"text\":\"晚上准备做什么？\"}"
```

### 9.4 查看 memory


## 10. 运行过程中会产生什么数据

运行后，项目会逐渐生成和更新记忆相关数据文件，常见包括：

- `memory/raw_log/`
- `memory/active_memory.jsonl`
- `memory/archive_memory.jsonl`
- `memory/core_memory.json`
- `memory/snapshots.jsonl`

其中：

- `Raw Log` 是原始运行数据
- `Core Memory` 是常驻上下文
- `Active Memory` 是当前活跃记忆
- `Snapshot` 用于恢复运行状态

## 12. 常见现状说明

- 没配模型 API 时，系统仍然能跑，但效果主要依赖 heuristic fallback
- `semantic embedding` 已经接好，`emotional embedding` 还没有
- `Archive Memory` 目前还是最小过渡实现，不是 README 理想完整版
- 当前主调试入口是 API，不是完整前端界面
- `read_url` 和 `search_web` 是当前内置的两个 Real Zone 能力
