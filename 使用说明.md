# Amadeus 使用说明

## 1. 项目概览

Amadeus 是一个以 persona 为核心、可持续运行的 Agent Runtime。

当前版本已经具备这几条主链路：

- persona 创建、切换、删除与 `soul.md` 编辑
- 日计划生成、执行、执行后 `replan`
- 用户消息进入交互层，角色回复后进入对话冷却期
- 角色在执行过程中触发主动联系，进入聊天链路
- 运行结果写入 memory，并可通过 API 和前端工作台查看

当前默认前端入口已经是集成工作台，而不是旧的独立实验页。

## 3. 配置 `.env`

先复制 [.env.example](/c:/Users/lenovo/Desktop/agent/Amadeus/.env.example) 为 `.env`，再按实际模型与工具情况填写。

### 3.1 模型路由

系统默认区分 4 个模型角色：

- `dialogue`
- `executor`
- `decision`
- `memory`

常见环境变量：

```dotenv
AMADEUS_DIALOGUE_PROVIDER=
AMADEUS_DIALOGUE_MODEL=
AMADEUS_DIALOGUE_API_KEY=
AMADEUS_DIALOGUE_BASE_URL=

AMADEUS_EXECUTOR_PROVIDER=
AMADEUS_EXECUTOR_MODEL=
AMADEUS_EXECUTOR_API_KEY=
AMADEUS_EXECUTOR_BASE_URL=

AMADEUS_DECISION_PROVIDER=
AMADEUS_DECISION_MODEL=
AMADEUS_DECISION_API_KEY=
AMADEUS_DECISION_BASE_URL=

AMADEUS_MEMORY_PROVIDER=
AMADEUS_MEMORY_MODEL=
AMADEUS_MEMORY_API_KEY=
AMADEUS_MEMORY_BASE_URL=
```

建议：

- `executor` 使用支持稳定结构化输出和 tool calling 的模型
- `decision` / `memory` 也建议使用能稳定返回结构化结果的模型
- `dialogue` 优先选更贴 persona 的模型

### 3.2 Embedding

语义向量有单独配置项，不要默认跟 `memory` 的文本模型共用地址。

关键变量：

```dotenv
AMADEUS_MEMORY_SEMANTIC_EMBEDDING_PROVIDER=
AMADEUS_MEMORY_SEMANTIC_EMBEDDING_MODEL=
AMADEUS_MEMORY_SEMANTIC_EMBEDDING_API_KEY=
AMADEUS_MEMORY_SEMANTIC_EMBEDDING_BASE_URL=
```

如果你使用阿里 DashScope 兼容接口，通常需要显式填写：

```dotenv
AMADEUS_MEMORY_SEMANTIC_EMBEDDING_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

### 3.3 Memory 检索开关

```dotenv
AMADEUS_MEMORY_RETRIEVAL_SEMANTIC_ENABLED=true
AMADEUS_MEMORY_RETRIEVAL_BM25_ENABLED=true
AMADEUS_MEMORY_RETRIEVAL_EMOTIONAL_ENABLED=false
AMADEUS_MEMORY_RETRIEVAL_RERANKER_ENABLED=true
AMADEUS_MEMORY_ACTIVE_RETENTION_DAYS=999
```

### 3.4 执行与交互相关参数

```dotenv
AMADEUS_EXECUTION_MAX_INNER_LOOP_TURNS=7
AMADEUS_EXECUTION_LOOP_PRE_REPLAN_BUFFER_SECONDS=30
AMADEUS_EXECUTION_GRANULARITY=minute
AMADEUS_INTERACTION_COOLDOWN_SECONDS=180
```

说明：

- `AMADEUS_INTERACTION_COOLDOWN_SECONDS` 默认是 `180` 秒，也就是 3 分钟
- 入站消息和角色主动外发消息都会进入同一套冷却期
- 冷却期内如果有新消息，会继续对话并重置冷却期
- 冷却期结束仍无新消息，会触发一次普通 `replan`

### 3.5 MCP

如果需要接本地或远程 MCP 服务，可配置：

```dotenv
AMADEUS_MCP_SERVERS_JSON=[{"server_id":"xiaohongshu","transport":"streamable_http","url":"http://127.0.0.1:18060/mcp","timeout_seconds":60}]
```

注意：

- 这里只是注册配置
- 对应 MCP 服务本身仍然需要先启动

## 5. 前端工作台使用

集成工作台包含 4 个主界面。

### 5.1 角色

角色页用于：

- 创建角色
- 删除角色
- 设置活动角色
- 编辑当前角色的 `soul.md`

当前 persona 数据保存在 `memory/personas/` 下，每个角色拥有独立工作目录。

### 5.2 工作台

工作台用于查看和调试当前运行态：

- 当前活动角色
- 当前运行状态
- 虚拟时间
- 当前日计划
- 当前高亮计划项
- 每个计划项对应的执行记录

说明：

- 虚拟时间会在页面上实时显示
- 你可以手动设置新的虚拟时间并立即应用
- 正在执行时，页面会通过轮询刷新执行结果

### 5.3 聊天

聊天页用于：

- 用户主动给角色发消息
- 查看角色主动触达和普通回复

当前实现是单消息流展示，不区分多会话列表。

### 5.4 设置

设置页当前主要用于看状态，而不是做复杂系统配置：

- tools 状态
- MCP 状态
- 基础健康状态

### 5.5 调试信息

调试信息没有单独拆成产品外页面，而是做成了各主页面里的折叠面板。

如需继续使用旧的独立调试页：

- `/front/debug`
- `/front/executor-lab`

如需单独调 `plan / replan`，仍可使用独立 Plan Lab，见第 8 节。

## 7. 快速体验顺序

推荐按这个顺序体验：

1. 配好 `.env`
2. 启动 `uvicorn`
3. 打开 `http://127.0.0.1:8001`
4. 在“角色”页创建 persona，并编辑 `soul.md`
5. 将该 persona 设为活动角色
6. 在“工作台”页查看当前状态和日计划
7. 在“聊天”页给角色发消息
8. 观察角色回复、冷却期和后续 `replan`

## 9. 常见问题

### 9.1 工作台能打开，但没有内容

优先检查：

- 是否已经创建并激活 persona
- 模型路由是否填写完整
- `dialogue / executor / decision / memory` 是否都能实际调用

### 9.2 embedding 报错或检索异常

优先检查：

- `AMADEUS_MEMORY_SEMANTIC_EMBEDDING_MODEL`
- `AMADEUS_MEMORY_SEMANTIC_EMBEDDING_API_KEY`
- `AMADEUS_MEMORY_SEMANTIC_EMBEDDING_BASE_URL`

最常见问题是 embedding 没有单独设置 `BASE_URL`，结果错误继承到了文本模型地址。

### 9.3 MCP 显示未连接

优先检查：

- `.env` 里的 `AMADEUS_MCP_SERVERS_JSON` 是否写对
- 对应 MCP 服务是否真的已经启动
- 本地端口是否能访问

### 9.4 聊天结束后角色没有立刻继续生活流

先确认：

- 当前是否仍在对话冷却期
- `AMADEUS_INTERACTION_COOLDOWN_SECONDS` 是否被设得过长

默认值是 `180` 秒。
